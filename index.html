Over de GIP

Mijn gip is het maken van een led bord die verbonden is met een toetsenbord via een ps2 aansluiting. De tekst die je typt op het toetsenbord, verschijnt op het schermpje van het led bord. Dit alles zal worden bestuurd met een ardweeny.

logboek

Deze week heb ik mijn website gemaakt, het grootste probleem dat ik had was dat ik mijn “goeie” ideeën niet kon omzetten in realiteit. Mijn eerste site (degene die moest afzijn voor de deadline) trok dan ook op weinig. Ik hoop dit nog te kunnen veranderen tegen dat de presentatie eraan komt. Het ander probleem dat ik had was dat ik heb ondervonden dat je zonder planning niet erg ver komt voor een GIP. Eigenlijk om goed te zijn moet je alles minuut per minuut plannen.

Deze week heb ik mij toegelegd op mijn website die wel iets beter is maar nog niet zoals ik het wil. Echte problemen heb ik deze week niet gehad, ik heb ook nog wel wat ideeën om te doen met mijn website maar ik denk niet dat dat mooi zal zijn. Zo wil ik bv als achtergrond (foto) een foto van Londen bij nacht doen, ziet er mooi uit op foto maar ik denk niet dat het zal passen op mijn website. De reden dat ik Londen zou gebruiken is omdat we ook dit jaar op stage gaan naar Londen, info hierover vind je bij “stage”.

Ik heb mijn website nog verbetert deze week, dat was mijn grootste doelstelling voor deze week. Eerst was mijn website 1 one pager, ik vond dit goed totdat ik begon na te denken, het was niet gebruiksvriendelijk, en zeker niet handig ook voor mij. Er was teveel scroll werk en dat koste heel wat tijd. Daarom besloot ik om er 2 aparte pagina's van te maken. Dit is overzichtelijker en beter ingedeeld. Ik was niet blij met hoe hij eruit zag, “If you don’t like something, change it” –Maya Angelou-. Dus dat deed ik. De website begint eindelijk er deftig uit te zien (te laat voor de deadline wel). De ideeën die eerst “onhaalbaar” leken, lijken nu opeens niet zo onhaalbaar meer, als je echt iets wilt, haal je het meestal ook wel, heb ik gemerkt. Mijn grootste probleem was dat ik op veel punten (met de website) vast zat en ik geen zin meer had om verder te doen omdat dingen niet werkten zoals ik dat zou willen. Een concreet voorbeeld: ik probeer zoveel mogelijk met bootstrap te werken, bootstrap is een verzameling hulpmiddelen voor het maken van websites en web toepassingen. Concreet wil dit zeggen dat dat je het leven makkelijker maakt, maar dat heeft ook een CSS (opmaak file). Het probleem was dat ik al een groot deel had van mijn eigen website en mijn eigen opmaak. Nu moest ik daar bootstrap bij doen, mijn eerste gedacht was dan om gewoon de code van mijn opmaak in de opmaak te steken van de bootstrap. Dit zorgde echter voor heel veel problemen, mijn opmaak was volledig weg, was niks goed meer aan. Ik heb het dan uiteindelijk kunnen oplossen door met 2 opmaak files te werken. Enerzijds mijn eigen opmaak, anderzijds de opmaak van bootstrap. Dankzij dat, kan ik mijn eigen opmaak blijven gebruiken EN kan ik toch bootstrap gebruiken.
